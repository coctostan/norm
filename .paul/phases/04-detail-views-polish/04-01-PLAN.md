<objective>
## Goal
Add a project detail page showing full project state (roadmap phases, decisions, blockers, loop position) and surface blocker alerts on the dashboard level.

## Purpose
The dashboard cards give an overview but users need to drill into a project to see its full roadmap, phase breakdown, decisions, and blockers. This is the core feature of Phase 4 — making NORM useful for day-to-day project monitoring.

## Output
- New `/project/[id]` route with detail view
- Backend detail endpoint returning full parsed state
- Dashboard project cards link to detail pages
- Blocker projects visually surfaced at dashboard level
</objective>


## AC-2: Roadmap phase breakdown displayed
```gherkin
Given a project has roadmap data with multiple phases
When the detail page loads
Then each phase is shown with its number, name, status, plan count, and completion date
```

## AC-3: Decisions and blockers displayed
```gherkin
Given a project has decisions and/or active blockers in its parsed state
When the detail page loads
Then decisions are listed in a table (decision, rationale, date, status)

```

## AC-4: Dashboard cards link to detail page
```gherkin
Given the dashboard shows project cards

```

## AC-5: Blocker alert banner on dashboard
```gherkin
Given one or more projects have active blockers
When the dashboard loads
Then a summary alert appears above the card grid showing total blocked projects
```
