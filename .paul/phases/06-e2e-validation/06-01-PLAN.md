  <action>
    1. In backend/main.py lifespan:
       - After auto-registering projects from config, sync each one
       - Import sync_project_by_path from watcher module
       - For each newly registered project AND each existing project:
         Run sync_project_by_path(db, abs_path) to populate state data
       - This ensures dashboard shows full project details on first load

    2. In backend/registry.py register_project:
       - After successful create_project + save_config, trigger initial sync
       - Import sync_project_by_path from backend.watcher
       - Call sync_project_by_path(db, abs_path) after registration
       - This ensures API-registered projects show details immediately

    Avoid: Breaking existing watcher flow — this adds one-time sync, watcher handles ongoing.
    Avoid: Importing from registry in main or vice versa in ways that create circular imports.
  </action>
