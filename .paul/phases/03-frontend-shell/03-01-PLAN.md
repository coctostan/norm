<objective>
## Goal
Scaffold the SvelteKit frontend with shadcn-svelte and Tailwind, create a WebSocket client store using Svelte 5 $state runes, and build the dashboard page with project cards showing phase, loop position, progress, and last activity.

## Purpose
This is the user-facing surface of NORM — without it, the backend data has no visual representation. The dashboard is the core value proposition: see all PALS projects at a glance with live updates.

## Output
- SvelteKit app at `frontend/` with shadcn-svelte components and dark theme
- WebSocket client store that connects to backend `/ws` and maintains reactive project state
- Dashboard page at `/` with a responsive card grid showing all monitored projects
</objective>

<task type="auto">
  <name>Task 2: WebSocket client store with $state runes</name>
  <files>frontend/src/lib/stores/websocket.svelte.ts</files>
  <action>
    1. Create websocket.svelte.ts using Svelte 5 runes ($state, $derived):
       - `projects` state: DashboardProject[] (initialized empty)
       - `connected` state: boolean
       - `connect()` function: creates WebSocket to ws://localhost:5173/ws (proxied via vite)
       - On "initial_state" message: set projects array
       - On "project_updated" message: replace projects array (backend sends full list)
       - Auto-reconnect on close with exponential backoff (1s, 2s, 4s, max 30s)
       - `disconnect()` function for cleanup
    2. Export a singleton store instance
    3. Connect on +layout.svelte mount (onMount), disconnect on destroy

    Avoid: Socket.io — native WebSocket is sufficient for local use.
    Avoid: Per-project diff logic — backend sends full project list on every update.
    Avoid: Complex state management — $state runes handle reactivity natively.
  </action>
  <verify>
    With backend running: frontend connects to WebSocket, receives initial_state, projects populate.
    Console shows connection status. Stopping backend triggers reconnect attempts.
  </verify>
  <done>AC-2 and AC-3 satisfied: WebSocket connects, receives state, updates reactively on changes</done>
</task>
