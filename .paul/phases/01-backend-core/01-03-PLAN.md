<objective>
## Goal
Add async file watcher using watchfiles that monitors `.paul/` directories for registered projects and triggers automatic sync when STATE.md, ROADMAP.md, or PROJECT.md change.

## Purpose
Enables real-time state updates — the core value of NORM. Without the watcher, users must manually trigger POST /sync. With it, NORM detects changes within ~1 second and updates the cache automatically. This completes Phase 1 (Backend Core).

## Output
- `backend/watcher.py` — async file watcher module
- Updated `backend/main.py` — watcher lifecycle integration
- `tests/test_watcher.py` — watcher unit tests
</objective>

<task type="auto">
  <name>Task 1: Create file watcher module</name>
  <files>backend/watcher.py, backend/config.py</files>
  <action>
    Create `backend/watcher.py` with:

    1. **FileWatcher class** with async start/stop lifecycle:
       - `__init__()` — accepts a callable to get current project list from DB
       - `start()` — launches background watching task via asyncio.create_task
       - `stop()` — cancels the background task gracefully
       - `_watch_loop()` — main loop that:
         a. Gets current project list from DB (refreshes periodically, e.g., every 30s or on each iteration)
         b. Builds list of paths to watch: `{project_path}/.paul/` for each registered project
         c. Uses `watchfiles.awatch()` to detect changes
         d. Filters changes to only STATE.md, ROADMAP.md, PROJECT.md filenames
         e. On relevant change: calls sync function for the affected project

    2. **Sync callback**: Reuse the sync logic from registry.py — call `parse_project_state()` then `upsert_project_state()`/`upsert_project_phases()` and `update_project_sync_time()`. Create a standalone `sync_project_by_path()` function that takes a db connection and project path.

    3. **File filter**: Only trigger on files matching `STATE.md`, `ROADMAP.md`, `PROJECT.md`. Use filename comparison, not glob.

    4. **Debounce**: watchfiles has built-in debounce (default 1600ms). Keep the default or set to 1000ms for responsiveness.

    5. **Error handling**: Log and continue on sync errors (don't crash the watcher). Use Python logging.

    Add `watch_interval: int = 1000` to Settings in config.py for debounce_ms config.

    Avoid:
    - Don't use polling/manual file stat — watchfiles uses rust-based OS notifications
    - Don't import the FastAPI router or depend on HTTP layer — keep watcher independent
    - Don't create a new DB connection per change — use a connection from the pool/factory
  </action>
