<objective>
## Goal
Standing FastAPI server with SQLite database, project registry API, GitHub repo with CI/CD and branch protection — the complete foundation for NORM.

## Purpose
This is the foundation everything else builds on. Without a running server, database, project registry, and proper repo hygiene, nothing else in NORM can function. The registry is the entry point — it tells NORM which `.paul/` directories to monitor.

## Output
- `backend/` Python package with FastAPI app, SQLite database, and project registry
- `norm.yaml` config file for registering project paths
- GitHub repo with CI pipeline and branch protection
- Server runs on `localhost:8000` with health check endpoint
</objective>

<task type="checkpoint:human-action" gate="blocking">
  <name>Task 5: Set branch protection on GitHub</name>
  <what-built>GitHub repo with CI pipeline ready for branch protection</what-built>
  <how-to-verify>
    After the initial push with CI pipeline, configure branch protection:

    Option A — CLI (recommended):
    ```bash
    gh api repos/{owner}/norm/rulesets -X POST -f '{"name":"main-protection","target":"branch","enforcement":"active","conditions":{"ref_name":{"include":["refs/heads/main"],"exclude":[]}},"rules":[{"type":"pull_request","parameters":{"required_approving_review_count":0,"dismiss_stale_reviews_on_push":false,"require_code_owner_review":false,"require_last_push_approval":false,"required_review_thread_resolution":false}},{"type":"required_status_checks","parameters":{"strict_status_checks_policy":false,"required_status_checks":[{"context":"CI"}]}}]}'
    ```

    Option B — GitHub UI:
    1. Go to repo Settings → Rules → Rulesets
    2. New ruleset → Branch
    3. Target: main
    4. Enable "Require a pull request before merging" (0 approvals required)
    5. Enable "Require status checks to pass" → add "CI" check
    6. Do NOT enable: required approvals, signed commits, linear history

    Rules summary (solo dev friendly):
    - ✅ Require PR (no direct push to main)
    - ✅ Require CI to pass
    - ❌ No required approvals (solo dev)
    - ❌ No signed commits
    - ❌ No linear history requirement
  </how-to-verify>
  <resume-signal>Type "done" after setting up branch protection, or "skip" to defer</resume-signal>
