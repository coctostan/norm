<objective>
## Goal
Add graceful shutdown with signal handling, watcher recovery for missing/moved directories, and WebSocket keep-alive pings to the backend — making the server resilient to restarts, filesystem changes, and stale connections.

## Purpose
Phase 7 scope: production-grade resilience. The backend currently has no signal handling (ungraceful shutdown leaves watchers orphaned), no recovery when a watched .paul/ directory disappears, and no WebSocket ping/pong (stale connections accumulate). This plan hardens the backend so it can run reliably as a long-lived process.

## Output
- Modified `backend/main.py` — signal handling + graceful shutdown sequence
- Modified `backend/watcher.py` — directory existence checks + recovery on missing paths
- Modified `backend/websocket.py` — periodic ping/pong + stale connection cleanup
- New `tests/test_watcher_resilience.py` — tests for watcher recovery and graceful degradation
</objective>

## AC-1: Graceful Shutdown
```gherkin
Given the NORM backend is running with active watchers and WebSocket connections
When the process receives SIGTERM or SIGINT
Then the watcher stops cleanly, WebSocket connections are closed with 1001 (going away), and the process exits with code 0
```

## AC-2: Watcher Recovery on Missing Directory
```gherkin
Given the watcher is monitoring a project's .paul/ directory
When that directory is deleted or becomes inaccessible
Then the watcher logs a warning, skips that project, and continues watching remaining projects without crashing
```

## AC-3: WebSocket Ping/Pong Keep-Alive
```gherkin
Given a WebSocket client is connected to /ws
When 30 seconds pass without activity
Then the server sends a ping frame, and if no pong is received within 10 seconds, the connection is closed and cleaned up
```

    - Clean up failed connections after ping round

    In backend/main.py websocket_endpoint:
    - Replace the bare `await websocket.receive_text()` keep-alive loop with a timeout-based approach:
      - Use asyncio.wait_for with a 30s timeout on receive_text()

      - If send fails, break and disconnect
    - This handles both server-initiated keep-alive and client message receipt
