<objective>
## Goal
Add a dashboard summary endpoint, WebSocket server for real-time push updates, and wire the file watcher to broadcast state changes to connected WebSocket clients.

## Purpose
The frontend needs two things: (1) a single API call to get all projects with their current state for the dashboard view, and (2) real-time updates when `.paul/` files change so the UI stays current without polling. This is the core real-time capability that makes NORM a live monitoring tool rather than a static viewer.

## Output
- `GET /api/dashboard` endpoint returning all projects with inline state
- `WebSocket /ws` endpoint with connection management
- File watcher broadcasts state changes to all connected WebSocket clients
- Tests for both REST and WebSocket functionality
</objective>

## AC-1: Dashboard endpoint returns all projects with state
```gherkin
Given two registered projects with synced state
When GET /api/dashboard is called
Then response contains a list of all projects, each with inline state (milestone, phase, loop position, progress, blockers), and a total count
```

## AC-2: WebSocket connection and message flow
```gherkin
Given a running NORM server
When a client connects to /ws
Then the connection is accepted and the client receives an initial state snapshot
And when a project's state changes, the client receives a push message with the updated project state
```

## AC-3: WebSocket connection management
```gherkin
Given multiple WebSocket clients connected
When a client disconnects
Then it is removed from the connection pool without affecting other clients
And when a state change occurs, only remaining connected clients receive the broadcast
```

## AC-4: Watcher broadcasts via WebSocket
```gherkin
Given the file watcher detects a change to a .paul/ file
When the watcher syncs the project state to the database
Then it also broadcasts the updated state to all connected WebSocket clients as a JSON message
```
