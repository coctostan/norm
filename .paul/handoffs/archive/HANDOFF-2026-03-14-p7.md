## Decisions Made

| Decision | Rationale | Impact |
|----------|-----------|--------|
| load_config/save_config as plain functions | Matches existing config.py style, simple | Easy to test and call from multiple modules |
| save_config preserves YAML settings section | Don't clobber user settings on write | Settings survive project mutations |

| Auto-sync all projects on startup | Dashboard was showing names without details | Full project data available on first load |
| Auto-sync on API registration | Required manual /sync call before | Immediate state data after registration |
